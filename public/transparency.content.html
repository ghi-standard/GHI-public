Transparency & documentation

The Global HashCost Index (GHI) is designed as an open standard with publicly documented components: methodology, engine, API schemas and version history.

Methodology

The methodology describes how Bitcoin production costs are modelled (cost structure, general assumptions, limitations).

  • Public methodology GHI v1.0

API & schemas

GHI API v1.0 defines the structure of the published data (indicator, snapshot) with stable contracts across the whole v1.x series.

  • API summary (this instance)
  • Full API documentation (docs/api.md)

Engine & source code

The current public engine uses a Sandbox Engine (v0.3.0): values are demonstration only, but formats and API contracts are final for GHI v1.0.

  • GHI Engine repository (ghi-engine)
