Méthodologie GHI – Version 1.0

Standard ouvert pour mesurer le coût mondial de production du Bitcoin

Versions officielles

  • Méthodologie : v1.0.0
  • API publique : v1.0
  • Moteur : v0.3.0 (Sandbox)
  • Documentation complète : methodology_public_v1.md

Résumé méthodologique

La méthodologie GHI v1.0 définit un cadre uniforme, transparent et reproductible permettant de mesurer le coût global de production du Bitcoin.

  • Coûts exprimés en USD par BTC
  • Trois métriques fondamentales : min / avg / max cost
  • Indicateur global journalier à structure fixe
  • Snapshot multi-jour pour analyses institutionnelles
  • Moteur Sandbox utilisé pour stabiliser l’API

Principes fondateurs

  • Transparence totale : versionnage strict, documents publics
  • Neutralité : aucune dépendance à un mineur, pool ou fabricant
  • Non prédictif : mesure économique, pas un modèle de prix
  • Reproductibilité : hypothèses publiées, audit possible
  • Stabilité institutionnelle : API garantie compatible lors du passage au moteur réel

Architecture méthodologique

La méthodologie GHI repose sur trois étages :

  1. Paramètres réseau : difficulté, hashrate, récompense de bloc
  2. Modélisation des coûts :
    • mix ASIC (efficacités, W/TH, amortissements)
    • prix régionaux de l’électricité (pondérés)
    • charges opérationnelles (OPEX)
  3. Agrégation globale :
    • coût minimum
    • coût moyen pondéré
    • coût maximum

GHI Methodology – Version 1.0

Open standard to measure the global cost of producing Bitcoin

Official Versions

  • Methodology: v1.0.0
  • Public API: v1.0
  • Engine: v0.3.0 (Sandbox)
  • Full documentation: methodology_public_v1.md

Overview

GHI v1.0 defines a consistent, transparent and reproducible framework to measure the global cost of producing Bitcoin.

  • Costs expressed in USD per BTC
  • Three key metrics: min / avg / max cost
  • Daily global indicator with fixed structure
  • Multi-day snapshot for institutional analysis
  • Sandbox Engine used to stabilise the public API

Core Principles

  • Transparency: strict versioning, public documentation
  • Neutrality: no dependency on miners, pools or manufacturers
  • Non-predictive: economic measurement, not a price model
  • Reproducibility: published assumptions, auditable design
  • Institutional Stability: API compatibility preserved as real engine is deployed

Methodological Architecture

The methodology follows a three-layer structure:

  1. Network parameters: difficulty, hashrate, block reward
  2. Cost modelling:
    • ASIC mix (efficiencies, W/TH, amortisation)
    • regional electricity pricing
    • operational expenditures (OPEX)
  3. Global aggregation:
    • minimum cost
    • weighted average cost
    • maximum cost
