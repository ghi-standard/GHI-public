    standard/GHI-Data-Model-v1.0.md

    transparency.html

La version GHI v1.0 constitue la première base stable du standard (data model, API publique, méthodologie, moteur sandbox). Toute évolution majeure (v2.x, v3.x, etc.) fera l’objet :

  • d’une mise à jour de la documentation dans les dépôts publics ;
  • d’un versionnage explicite des composants (API, méthodologie, moteur) ;
  • d’un historique traçable via les Pull Requests et changelogs publics.

GHI Technical Standards Hub – v1.0

This page centralises the main technical documents for the Global HashCost Index (GHI) version v1.0. It is intended for technical teams, analysts and institutions that want to integrate or audit the standard.

1. GHI Standard and Data Model

The core of the GHI v1.0 standard is defined by two documents:

  • GHI Technical Standard v1.0 – formal description of the standard (structures, contracts, high-level logic):
    standard/GHI-Technical-Standard-v1.0.md
  • GHI Data Model v1.0 – official data schema (Snapshot, Region, HistoryPoint, etc.):
    standard/GHI-Data-Model-v1.0.md

2. Methodology and Transparency

The economic principles and transparency framework are documented through:

  • GHI Methodology v1.0.0 – public overview (FR/EN):
    methodology.html
  • Transparency v1.0 – scope, limits and governance of transparency:
    transparency.html
  • Public Methodology v1.0 – detailed reference (English, engine repo):
    ghi-engine/docs/methodology_public_v1.md

3. GHI Public API v1.0

API integrators can rely on the following resources:

  • API overview page:
    api.html
  • API Quickstart v1.0 – ready-to-use call examples:
    api-quickstart.html
  • API Reference v1.0 – endpoint details and parameters:
    api-reference.html
  • API Specification (technical format) – engine repository:
    ghi-engine/docs/api.md

4. GHI Engine – design and versioning

The GHI Engine is documented at design and versioning level:

  • Engine Design Overview (Sandbox Engine v0.4.0):
    ghi-engine/docs/ENGINE_DESIGN.md
  • Versioning Model (API / Methodology / Engine):
    ghi-engine/docs/VERSIONS.md
  • Public engine repository (sandbox):
    github.com/ghi-standard/ghi-engine

5. Regions and regional model

The regional model is presented in the dedicated pages:

  • GHI Regions v1.0 – major geographic areas and weighting principles:
    regions.html
  • Governance and update process:
    governance.html

6. Institutional documents

For institutions, supervisors and large actors:

  • GHI Whitepaper v1.0 (EN/FR) – bilingual Markdown version:
    public/docs/WHITEPAPER_GHI_v1.0.md
  • GHI Institutional Pack – Slide Deck v1.0 (20 slides, EN/FR):
    public/docs/GHI_Institutional_Pack_SlideDeck_v1.0.md

7. Scope of version 1.0 and future evolution

The GHI v1.0 release is the first stable baseline of the standard (data model, public API, methodology, sandbox engine). Any major evolution (v2.x, v3.x, etc.) will involve:

  • updated documentation in the public repositories;
  • explicit versioning of each component (API, methodology, engine);
  • a fully traceable history via public Pull Requests and changelogs.
