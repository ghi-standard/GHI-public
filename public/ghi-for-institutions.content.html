GHI for Institutions

The Global HashCost Index (GHI) is an open standard measuring Bitcoin production cost at global scale. It is designed from day one for central banks, supervisors, regulated financial institutions, listed companies and audit firms.

1. What is GHI used for?

  • Providing an independent reference for the cost of producing Bitcoin (min / avg / max).
  • Enabling comparability across regions, time periods and energy price scenarios.
  • Offering an additional risk indicator alongside market data (price, volatility, volumes).
  • Supporting prudential supervision, sensitivity analysis and stress-testing frameworks.

2. Institutional use cases

  • Central banks & supervisors
    — monitoring mining economic sustainability,
    — macro-energy scenarios,
    — financial stability assessments.
  • Asset managers & funds
    — measuring miner capitulation risk,
    — integrating production cost into valuation models,
    — due diligence on Bitcoin-exposed products.
  • Listed companies & miners
    — benchmarking cost structures against global / regional GHI,
    — ESG and annual report disclosures.
  • Audit, rating & advisory firms
    — neutral reference for business-model assessment,
    — standardized documentation for risk committees.

3. What GHI provides

  • Public API – daily indicator and structured global snapshot (API documentation, Quickstart).
  • Regional breakdown – synthetic profiles (North America, Europe, Asia ex-China, …).
  • Methodology – economic model and assumptions (Methodology page).
  • Transparency & governance – versioning, limitations and governance framework (Transparency, Governance).
  • Technical notes – focused analytical studies (links available on the Governance page).

4. Public vs. private engine

The public ghi-engine repository exposes a Sandbox engine returning synthetic but stable data, allowing institutions to integrate the standard and test the API without disclosing internal assumptions.

The full engine implementation (real data, calibration parameters, ingestion pipelines) is maintained in a private repository, with controlled access procedures for authorities and institutional partners under NDA.

5. How to get started?

  1. Review the Methodology and Transparency pages.
  2. Test the API using the API Quickstart.
  3. Integrate the GHI snapshot into internal risk, research or ESG dashboards.
  4. Open a GitHub issue or Pull Request for questions or contributions.

6. Institutional & audit access

Institutions interested in deeper access (audits, specific scenarios, controlled access to the private engine) can reach out through the official GitHub channels. Dedicated procedures (NDA, secure environment, historical replays) are envisaged for supervisors and strategic partners.

GHI pour les institutions

Le Global HashCost Index (GHI) est un standard ouvert mesurant le coût de production du Bitcoin à l’échelle mondiale. Il est conçu dès l’origine pour les institutions financières, régulateurs, banques centrales, entreprises cotées et cabinets d’audit.

1. À quoi sert le GHI ?

  • Fournir une référence indépendante du coût de production du Bitcoin (min / moyen / max).
  • Permettre la comparabilité entre régions, périodes et scénarios énergétiques.
  • Offrir un indicateur de risque complémentaire aux données de marché.
  • Servir de base à la supervision prudentielle et aux analyses de stress.

2. Cas d’usage institutionnels

  • Banques centrales & régulateurs — stabilité financière, scénarios macro-énergétiques.
  • Gestionnaires d’actifs & fonds — valorisation, due diligence, risque minier.
  • Entreprises cotées & miners — benchmark de coûts, reporting ESG.
  • Cabinets d’audit et de conseil — analyse neutre et documentation standardisée.

3. Ce que fournit GHI

  • API publique – indicateur quotidien et snapshot global structuré.
  • Découpage régional – profils synthétiques comparables.
  • Méthodologie – description du modèle économique.
  • Transparence & gouvernance – cadre de référence et limites.
  • Notes techniques – analyses ciblées et études approfondies.

4. Moteur public vs. moteur privé

Le dépôt public ghi-engine expose un moteur Sandbox fournissant des données synthétiques mais stables, permettant de tester l’API sans divulguer les hypothèses internes.

Le moteur complet (données réelles, calibrations, pipelines) est maintenu dans un dépôt privé avec des procédures d’accès dédiées aux autorités et partenaires institutionnels.
