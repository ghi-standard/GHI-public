API GHI v1.0 – Référence détaillée

Cette page décrit la référence de l’API publique du Global HashCost Index (GHI), version v1.0. Toutes les réponses sont générées par le Sandbox Engine v0.4.0-sandbox, qui fournit des données factices mais stables pour la démonstration et les tests d’intégration.

1. Base URL et version

L’API GHI v1.0 est exposée sous un préfixe de base, suivi des endpoints stables de la version 1 : 

Base path : /v1/ghi

Exemples d’URL complètes (selon l’environnement de déploiement) : 

https://sandbox.globalhashcostindex.org/v1/ghi/indicator
https://sandbox.globalhashcostindex.org/v1/ghi/snapshot

Les chemins documentés ci-dessous sont indépendants du nom de domaine et restent valables tant que l’API publique GHI v1.0 est active.

2. Formats et en-têtes

  • Format de réponse : JSON UTF-8.
  • En-tête recommandé : Accept: application/json.
  • Authentification : aucune dans le Sandbox public v0.4.0. Des mécanismes d’accès restreint (clé API, OAuth2, etc.) pourront être introduits dans des déploiements institutionnels dédiés.

3. Métadonnées de version (réponses)

Toutes les réponses GHI v1.0 incluent un bloc de métadonnées permettant d’identifier précisément la version de l’API, la méthodologie et le moteur utilisés : 

{
  "api_version": "v1.0",
  "methodology_version": "1.0.0",
  "engine_version": "v0.4.0-sandbox",
  "data_origin": "sandbox",
  "stability_seed": 42
}
  • api_version : version de l’API publique (v1.0).
  • methodology_version : version de la méthodologie GHI (1.0.0).
  • engine_version : version du moteur utilisé (Sandbox Engine v0.4.0-sandbox).
  • data_origin : nature de la source de données (ici "sandbox" pour indiquer qu’il s’agit d’un jeu synthétique).
  • stability_seed : identifiant technique garantissant la stabilité du jeu de données factices.

4. Endpoint : GET /v1/ghi/indicator

4.1. Description

Renvoie l’indicateur quotidien global du coût de production du Bitcoin, sous la forme d’un triplet min / avg / max pour une date donnée. Dans le Sandbox v0.4.0, les valeurs sont générées de manière déterministe mais ne reflètent pas les coûts réels.

4.2. Requête

GET /v1/ghi/indicator HTTP/1.1
Host: sandbox.globalhashcostindex.org
Accept: application/json

4.3. Paramètres de requête

Paramètre Type Obligatoire Description
date string (YYYY-MM-DD) Non Date de l’indicateur souhaité. Si omis, le Sandbox renvoie la dernière date disponible pour le jeu de données factices.

4.4. Exemple avec curl

curl "https://sandbox.globalhashcostindex.org/v1/ghi/indicator"

4.5. Exemple de réponse (simplifiée)

{
  "date": "2025-12-08",
  "currency": "USD",
  "min_cost": 38000.12,
  "avg_cost": 41000.45,
  "max_cost": 44000.89,
  "api_version": "v1.0",
  "methodology_version": "1.0.0",
  "engine_version": "v0.4.0-sandbox",
  "data_origin": "sandbox",
  "stability_seed": 42
}

Dans le Sandbox Engine v0.4.0, le champ currency est systématiquement égal à "USD".

4.6. Codes de réponse

  • 200 OK : indicateur retourné avec succès.
  • 400 Bad Request : paramètres invalides (ex. format de date incorrect).
  • 404 Not Found : aucune donnée disponible pour la date demandée (peut dépendre de la configuration du Sandbox).
  • 500 Internal Server Error : erreur interne inattendue.

4.7. Exemple d’erreur JSON

{
  "detail": "Invalid date format, expected YYYY-MM-DD.",
  "api_version": "v1.0"
}

5. Endpoint : GET /v1/ghi/snapshot

5.1. Description

Renvoie un snapshot structuré du coût de production du Bitcoin, comprenant un indicateur global et un détail par région, conformément au GHI Data Model v1.0. Le contenu précis des objets (champs régionaux, énergie, machines, etc.) est défini dans la spécification de données.

5.2. Requête

GET /v1/ghi/snapshot HTTP/1.1
Host: sandbox.globalhashcostindex.org
Accept: application/json

5.3. Paramètres de requête

Paramètre Type Obligatoire Description
date string (YYYY-MM-DD) Non Date du snapshot souhaité. Si omis, le Sandbox renvoie la dernière date disponible pour le jeu de données factices.

5.4. Exemple avec curl

curl "https://sandbox.globalhashcostindex.org/v1/ghi/snapshot"

5.5. Exemple de réponse (structure générale)

{
  "date": "2025-12-08",
  "currency": "USD",
  "indicator": {
    "min_cost": 38000.12,
    "avg_cost": 41000.45,
    "max_cost": 44000.89
  },
  "regions": [
    {
      "region_id": "north_america",
      "name": "North America",
      "hashrate_pct": 0.35,
      "min_cost": 39000.10,
      "avg_cost": 41500.25,
      "max_cost": 44500.90
      // ... autres champs définis dans le Data Model v1.0
    }
    // ... autres régions
  ],
  "api_version": "v1.0",
  "methodology_version": "1.0.0",
  "engine_version": "v0.4.0-sandbox",
  "data_origin": "sandbox",
  "stability_seed": 42
}

La structure complète des objets regions (coût, énergie, machines, métadonnées) est définie dans le GHI Data Model v1.0. Le Sandbox Engine respecte ce modèle, mais avec des valeurs purement synthétiques.

5.6. Codes de réponse

  • 200 OK : snapshot renvoyé avec succès.
  • 400 Bad Request : paramètres invalides.
  • 404 Not Found : aucune donnée disponible pour la date demandée.
  • 500 Internal Server Error : erreur interne inattendue.

6. Gestion des versions et compatibilité

  • L’API GHI v1.0 est conçue pour rester stable : les endpoints, les noms de champs et les types principaux ne changent pas sans nouvelle version majeure.
  • Les évolutions mineures (ex. champs supplémentaires non obligatoires) peuvent être introduites avec des numéros de patch, en conservant la compatibilité ascendante.
  • Les futures versions (v2.x, v3.x, …) seront exposées sur d’autres préfixes (/v2/ghi, etc.) afin de ne pas casser les intégrations existantes.

7. Références

  • GHI Technical Standard v1.0
  • Guide de démarrage rapide API GHI v1.0

GHI API v1.0 – Detailed Reference

This page describes the public API reference for the Global HashCost Index (GHI), version v1.0. All responses are produced by the Sandbox Engine v0.4.0-sandbox, which serves synthetic but stable data for demonstration and integration tests.

1. Base URL and version

The GHI v1.0 API is exposed under a stable base path, followed by the versioned endpoints: 

Base path: /v1/ghi

Examples of full URLs (deployment-dependent):

https://sandbox.globalhashcostindex.org/v1/ghi/indicator
https://sandbox.globalhashcostindex.org/v1/ghi/snapshot

The paths documented below are independent from the hostname and remain valid as long as the GHI public API v1.0 is active.

2. Formats and headers

  • Response format: JSON UTF-8.
  • Recommended header: Accept: application/json.
  • Authentication: none for the public Sandbox v0.4.0. Restricted-access deployments (API keys, OAuth2, etc.) may be used in institutional environments.

3. Version metadata (responses)

All GHI v1.0 responses include a metadata block identifying the API, methodology and engine versions used: 

{
  "api_version": "v1.0",
  "methodology_version": "1.0.0",
  "engine_version": "v0.4.0-sandbox",
  "data_origin": "sandbox",
  "stability_seed": 42
}
  • api_version: public API version (v1.0).
  • methodology_version: GHI methodology version (1.0.0).
  • engine_version: engine version in use (Sandbox Engine v0.4.0-sandbox).
  • data_origin: nature of the data source (here "sandbox" to indicate synthetic data).
  • stability_seed: technical identifier ensuring stability of the synthetic dataset.

4. Endpoint: GET /v1/ghi/indicator

4.1. Description

Returns the daily global indicator of Bitcoin production cost, as a min / avg / max triplet for a given date. In Sandbox v0.4.0, values are deterministic but do not reflect real-world costs.

4.2. Request

GET /v1/ghi/indicator HTTP/1.1
Host: sandbox.globalhashcostindex.org
Accept: application/json

4.3. Query parameters

Parameter Type Required Description
date string (YYYY-MM-DD) No Target date for the indicator. If omitted, the Sandbox returns the last available date in the synthetic dataset.

4.4. Example using curl

curl "https://sandbox.globalhashcostindex.org/v1/ghi/indicator"

4.5. Example response (simplified)

{
  "date": "2025-12-08",
  "currency": "USD",
  "min_cost": 38000.12,
  "avg_cost": 41000.45,
  "max_cost": 44000.89,
  "api_version": "v1.0",
  "methodology_version": "1.0.0",
  "engine_version": "v0.4.0-sandbox",
  "data_origin": "sandbox",
  "stability_seed": 42
}

In Sandbox Engine v0.4.0, the currency field is always set to "USD".

4.6. Response codes

  • 200 OK: indicator successfully returned.
  • 400 Bad Request: invalid parameters (e.g. wrong date format).
  • 404 Not Found: no data available for the requested date (behaviour may depend on Sandbox configuration).
  • 500 Internal Server Error: unexpected internal error.

4.7. Error JSON example

{
  "detail": "Invalid date format, expected YYYY-MM-DD.",
  "api_version": "v1.0"
}

5. Endpoint: GET /v1/ghi/snapshot

5.1. Description

Returns a structured snapshot of Bitcoin production costs, including a global indicator and a regional breakdown, as defined in the GHI Data Model v1.0. The exact content of regional objects (cost, energy, hardware, metadata, etc.) is defined in the data model specification.

5.2. Request

GET /v1/ghi/snapshot HTTP/1.1
Host: sandbox.globalhashcostindex.org
Accept: application/json

5.3. Query parameters

Parameter Type Required Description
date string (YYYY-MM-DD) No Target date for the snapshot. If omitted, the Sandbox returns the last available date in the synthetic dataset.

5.4. Example using curl

curl "https://sandbox.globalhashcostindex.org/v1/ghi/snapshot"

5.5. Example response (overall structure)

{
  "date": "2025-12-08",
  "currency": "USD",
  "indicator": {
    "min_cost": 38000.12,
    "avg_cost": 41000.45,
    "max_cost": 44000.89
  },
  "regions": [
    {
      "region_id": "north_america",
      "name": "North America",
      "hashrate_pct": 0.35,
      "min_cost": 39000.10,
      "avg_cost": 41500.25,
      "max_cost": 44500.90
      // ... additional fields as defined in Data Model v1.0
    }
    // ... other regions
  ],
  "api_version": "v1.0",
  "methodology_version": "1.0.0",
  "engine_version": "v0.4.0-sandbox",
  "data_origin": "sandbox",
  "stability_seed": 42
}

The full structure of regions objects (cost, energy, hardware, metadata) is defined in GHI Data Model v1.0. The Sandbox Engine follows this model but uses synthetic values only.

5.6. Response codes

  • 200 OK: snapshot successfully returned.
  • 400 Bad Request: invalid parameters.
  • 404 Not Found: no data available for the requested date.
  • 500 Internal Server Error: unexpected internal error.

6. Versioning and compatibility

  • The GHI v1.0 API is designed to remain stable: endpoints, field names and main types will not change without a new major version.
  • Minor evolutions (e.g. additional optional fields) may be introduced using patch numbers while preserving backward compatibility.
  • Future versions (v2.x, v3.x, …) will be exposed on separate prefixes (/v2/ghi, etc.) to avoid breaking existing integrations.
