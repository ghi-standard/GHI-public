Sandbox Engine v0.4.2 – Moteur de démonstration GHI

1. Rôle du Sandbox Engine

Le Sandbox Engine v0.4.2 est le moteur de démonstration utilisé pour exposer l’API publique GHI. Il ne calcule pas les coûts réels de production, mais fournit un comportement stable, déterministe et reproductible pour les intégrations techniques, les POC et les tests automatiques.

Le moteur “sandbox” est séparé du futur moteur de production institutionnel. Il illustre la structure des réponses, le modèle de données et le cycle de version.

Principales caractéristiques

  • Modèle de données aligné sur l’API publique v1.1.
  • Régions synthétiques : us, eu, cn.
  • Statistiques globales (min / avg / max) calculées à partir des régions.
  • Historique synthétique minimal pour les tests d’intégration.
  • Métriques réseau et forecast exposées en mode “beta”.
  • Séparation stricte entre moteur public et moteur interne.
  • Architecture modulaire : loader → cost engine → aggregator → API.
  • Support complet de l’API publique v1.1 (snapshot, history, regions, metadata, network, forecast).
  • Version explicite : v0.4.2-sandbox.

2. Endpoints couverts par le moteur sandbox

Le moteur alimente les endpoints suivants, en version v1.1 :

  • /v1/ghi/snapshot – Coût de production synthétique par région + statistiques globales.
  • /v1/ghi/history – Historique synthétique (quelques points temporels pour tests).
  • /v1/ghi/regions – Liste des régions supportées par le modèle sandbox.
  • /v1/ghi/metadata – Métadonnées sur l’API et la version du moteur.
  • /v1/ghi/network – Métriques réseau synthétiques (beta).
  • /v1/ghi/forecast – Prévision simple du coût moyen (beta).

Toutes les réponses incluent les champs standardisés version, timestamp_utc, engine_sandbox_version.

3. Données synthétiques et limites

Le Sandbox Engine n’a aucune vocation d’estimation réelle du coût de production. Les chiffres sont volontairement simplifiés :

  • Valeurs codées en dur dans le code (sans accès à des sources externes).
  • Pas de raccordement à des tarifs d’électricité réels ni à des mix machines.
  • Historique généré à partir de règles simples (tendance, variation contrôlée).

L’objectif est de fournir un environnement prévisible pour :

  • intégrer l’API GHI dans des systèmes existants ;
  • écrire des tests automatiques robustes ;
  • préparer l’arrivée de futures données institutionnelles réelles.

4. Gouvernance et évolutions

La version v0.4.2-sandbox est alignée sur :

  • l’API publique v1.1 (structure de réponses unifiée) ;
  • le modèle de régions et de métadonnées décrit dans la documentation technique ;
  • les tests automatiques publics présents sur le dépôt GitHub ghi-engine.

Les futures versions du moteur (v0.5.x, v1.0.0, …) suivront une gouvernance publiquement documentée dans les fichiers ENGINE_DESIGN.md et VERSIONS.md.

Sandbox Engine v0.4.2 – GHI Demo Engine

1. Role of the Sandbox Engine

Sandbox Engine v0.4.2 is the demo engine powering the public GHI API. It does not compute real-world production costs; instead, it provides a stable, deterministic and reproducible behaviour for technical integrations, POCs and automated tests.

The sandbox engine is kept separate from the future institutional production engine. It illustrates the response structure, data model, and version lifecycle.

Key characteristics

  • Data model aligned with public API v1.1.
  • Synthetic regions: us, eu, cn.
  • Global statistics (min / avg / max) computed from regional values.
  • Minimal synthetic history for integration testing.
  • Network metrics and forecast endpoints exposed in “beta” mode.
  • Strict separation between public and private engines.
  • Modular architecture (loader → cost engine → aggregator → API).
  • Full support of public API v1.1 (snapshot, history, regions, metadata, network, forecast).
  • Explicit version: v0.4.2-sandbox.

2. Endpoints powered by the sandbox engine

The engine feeds the following v1.1 endpoints:

  • /v1/ghi/snapshot – Synthetic production costs per region + global statistics.
  • /v1/ghi/history – Synthetic time series (few points for tests).
  • /v1/ghi/regions – Regions supported by the sandbox model.
  • /v1/ghi/metadata – API and engine metadata.
  • /v1/ghi/network – Synthetic network metrics (beta).
  • /v1/ghi/forecast – Simple average-cost forecast (beta).

All responses include standardized fields version, timestamp_utc, and engine_sandbox_version.

3. Synthetic data & limitations

The sandbox engine has no ambition to reflect real production costs. Numbers are deliberately simplified:

  • Values are hard-coded in the source code (no external data sources).
  • No real electricity tariffs, ASIC fleet or regional energy mix.
  • History is generated from simple rules (trend + controlled variation).

The goal is to provide a predictable environment for:

  • integrating the GHI API into existing systems;
  • writing robust automated tests;
  • preparing the arrival of future institutional-grade data.

4. Governance & evolution

v0.4.2-sandbox is aligned with:

  • public API v1.1 (unified response format);
  • regional and metadata models defined in the technical documentation;
  • public automated tests in the ghi-engine GitHub repository.

Future engine versions (v0.5.x, v1.0.0, …) will follow a documented governance model described in ENGINE_DESIGN.md and VERSIONS.md.
