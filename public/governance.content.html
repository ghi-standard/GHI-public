Gouvernance du standard GHI

Le Global HashCost Index est un standard ouvert. Sa gouvernance repose sur la transparence, le versionnage et la publication systématique des changements.

Principes

  • API, méthodologie et moteur ont chacun leur version.
  • Les changements majeurs créent une nouvelle version majeure (v2.0, v3.0…).
  • Tous les changements passent par des Pull Requests publiques.

Processus

  • Proposition : issue ou Pull Request GitHub.
  • Revue : examen technique, méthodologique et d’intégration.
  • Validation : merge + création d’une release GitHub.

Releases officielles : github.com/ghi-standard/ghi-engine/releases

GHI Governance

The Global HashCost Index is an open standard. Governance relies on transparency, versioning and public change history.

Principles

  • API, methodology and engine each have their own versions.
  • Breaking changes trigger new major versions (v2.0, v3.0…).
