Gouvernance du standard GHI

Le Global HashCost Index (GHI) est un standard ouvert dédié à l’analyse du coût de production du Bitcoin. Sa gouvernance repose sur la transparence, la traçabilité et un processus de publication strictement contrôlé.

Principes clés

  • Le standard GHI est développé publiquement sur GitHub.
  • Toute évolution passe par des issues et des Pull Requests publiques.
  • L’API, la méthodologie et le moteur disposent de versions indépendantes.
  • Les snapshots publiés sont immuables et ne sont jamais modifiés a posteriori.
  • Toute correction ou amélioration donne lieu à la publication d’un nouveau snapshot.

Processus de gouvernance

  • Proposition : issue ou Pull Request publique.
  • Revue : analyse technique, méthodologique et d’intégration.
  • Validation : merge et publication officielle versionnée.

Cadre de validation

Le standard GHI est conçu pour être évalué par des tiers indépendants (chercheurs, analystes, institutions) à partir de ses éléments publics.

  • La validation porte sur la méthodologie, la structure des données et la traçabilité.
  • Les données sources privées ne font pas partie du périmètre de validation publique.
  • Aucune validation ne constitue une approbation commerciale ou institutionnelle.
  • Les retours peuvent être formulés via GitHub ou par écrit.

GHI Governance

The Global HashCost Index (GHI) is an open standard dedicated to Bitcoin production cost analysis. Its governance relies on transparency, traceability and a strictly controlled publication process.

Key principles

  • The GHI standard is developed publicly on GitHub.
  • All changes go through public issues and Pull Requests.
  • The API, methodology and engine maintain independent versioning.
  • Published snapshots are immutable and never altered retroactively.
  • Any correction or improvement results in a new snapshot publication.

Governance process

  • Proposal: public GitHub issue or Pull Request.
  • Review: technical, methodological and integration review.
  • Validation: merge and official versioned release.

Validation framework

The GHI standard is designed to be assessed by independent third parties using only its publicly available components.

  • Validation focuses on methodology, data structure and traceability.
  • Private source data is explicitly excluded from public validation.
  • No validation implies commercial or institutional endorsement.
  • Feedback may be submitted via GitHub or in written form.
