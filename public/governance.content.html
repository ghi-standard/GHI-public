Gouvernance du standard GHI

Le Global HashCost Index est un standard ouvert. Sa gouvernance repose sur la transparence, le versionnage et la publication systématique des évolutions.

Principes

  • L’API, la méthodologie et le moteur possèdent chacun leur version indépendante.
  • Tout changement majeur crée une nouvelle version majeure (v2.0, v3.0…).
  • Toutes les modifications passent par des Pull Requests publiques GitHub.

Processus officiel

  • Proposition : création d’une issue ou Pull Request GitHub.
  • Revue : examen technique, méthodologique et d’intégration.
  • Validation : merge + création d’une release officielle.

GHI Governance

The Global HashCost Index is an open standard. Governance relies on transparency, versioning and a public change history.

Principles

  • The API, methodology and engine each maintain their own version.
  • Breaking changes require a new major version (v2.0, v3.0…).
  • All modifications go through public GitHub Pull Requests.

Official process

  • Proposal: GitHub issue or Pull Request.
  • Review: technical, methodological and integration checks.
  • Validation: merge + creation of an official release.
