# Politique de sécurité / Security Policy

## 1. Objet / Purpose

**FR**  
Ce dépôt décrit le standard ouvert « Global HashCost Index (GHI) ».  
La présente politique explique comment signaler une faille de sécurité potentielle et comment nous gérons ces signalements.

**EN**  
This repository documents the open standard “Global HashCost Index (GHI)”.  
This policy explains how to report potential security issues and how we handle such reports.

---

## 2. Portée / Scope

**FR**  
La politique de sécurité couvre :

- Le dépôt GitHub `ghi-standard/GHI-public` (code HTML/Markdown, documentation).
- Les exemples de code et scripts fournis dans ce dépôt.
- Les liens et ressources officielles mentionnés dans cette documentation.

Elle **ne couvre pas** :

- Les implémentations tierces du standard GHI.
- Les systèmes et infrastructures de partenaires externes.
- Les environnements internes privés, non publiés sur GitHub.

**EN**  
This security policy covers:

- The `ghi-standard/GHI-public` GitHub repository (HTML/Markdown, documentation).
- Code samples and example scripts shipped with this repository.
- Official links and resources referenced by this documentation.

It **does not cover**:

- Third-party implementations of the GHI standard.
- Systems and infrastructure operated by external partners.
- Private internal environments not published on GitHub.

---

## 3. Signalement d'une vulnérabilité / Reporting a vulnerability

**FR**  
Si vous pensez avoir identifié une vulnérabilité liée à ce dépôt :

1. **Ne publiez pas** l’information dans un ticket public ou sur les réseaux sociaux.  
2. Préparez un rapport incluant :
   - Une description claire du problème.
   - Les étapes pour reproduire la vulnérabilité.
   - Tout élément de preuve technique (logs, captures d’écran, PoC) dans la mesure du raisonnable.
3. Envoyez ce rapport à l’adresse suivante :  
   **`security@ghi-standard.org`** (adresse de contact dédiée aux questions de sécurité).

Nous accusons réception du signalement dans un délai raisonnable, et reviendrons vers vous pour demander des clarifications si nécessaire.

**EN**  
If you believe you have found a security issue related to this repository:

1. **Do not disclose** the information in a public issue or on social media.  
2. Prepare a report including:
   - A clear description of the issue.
   - Steps to reproduce the vulnerability.
   - Any technical evidence (logs, screenshots, PoC) within reasonable limits.
3. Send your report to:  
   **`security@ghi-standard.org`** (dedicated contact for security matters).

We will acknowledge receipt of your report within a reasonable timeframe and may contact you for further clarification when needed.

## 4. Attentes mutuelles / Responsible disclosure

**FR**  
Nous demandons aux chercheurs en sécurité :

- D’appliquer les principes de **divulgation responsable**.
- De ne pas tenter d’exploiter une vulnérabilité au-delà de ce qui est strictement nécessaire pour la démontrer.
- De ne pas compromettre la confidentialité ou l’intégrité de données réelles.

En retour, nous nous engageons à :

- Analyser les signalements de bonne foi.
- Corriger les problèmes avérés dans un délai raisonnable.
- Mentionner et remercier (si souhaité) les chercheurs qui contribuent à l’amélioration de la sécurité du standard GHI.

**EN**  
We ask security researchers to:

- Follow **responsible disclosure** principles.
- Avoid exploiting any vulnerability beyond what is strictly necessary to demonstrate it.
- Refrain from compromising the confidentiality or integrity of real-world data.

In return, we commit to:

- Review good-faith reports.
- Fix confirmed issues within a reasonable timeframe.
- Acknowledge and thank (if desired) researchers who contribute to improving the security of the GHI standard.

## 5. Limites de responsabilité / Disclaimer

**FR**  
Les informations de ce dépôt sont fournies « en l’état ».  
Cette politique de sécurité ne constitue pas une garantie d’absence de vulnérabilité, ni un engagement juridique de résultat.

**EN**  
The information in this repository is provided “as is”.  
This security policy does not constitute a guarantee that the software or documentation is free from vulnerabilities, nor any legal commitment to a particular outcome.
