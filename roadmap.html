Roadmap & versions

Cette page présente les grandes étapes de développement de GHI et le principe de versionnage du modèle. La roadmap détaillée pourra être enrichie au fil du temps.

Version actuelle – GHI v1.0 (prévue)

  • Découpage régional de base (Amérique du Nord, Europe, Asie, etc.).
  • Moteur Python de calcul du coût par région et global.
  • API FastAPI avec endpoints principaux (snapshot, régions, historique, stats).
  • Dashboard web simple, basé sur des appels à l’API.
  • Documentation de méthodologie et de gouvernance.

Étapes suivantes (exemples)

v1.x – Améliorations incrémentales

  • Affinage des sources de données énergétiques par région.
  • Meilleure modélisation du mix machines par région.
  • Amélioration de la couverture temporelle de l’historique.
  • Ajout de métriques dérivées (par exemple coût / TH, comparaisons avec le prix spot).

v2.0 – Extension de périmètre

  • Segmentation régionale plus fine, si les données le permettent.
  • Ouverture à davantage de partenaires pour la fourniture de données.
  • Modules complémentaires possibles (par exemple composante GES, scénarios).

Changelog

Le détail des changements (changelog) pour chaque version de GHI pourra être publié dans un dépôt public, avec pour chaque entrée :

  • la version concernée,
  • la date de publication,
  • les modifications principales,
  • l’impact attendu sur les résultats.
